Day 2 - Wakanda UI Coding Intro





Many similarities in the JavaScript you write for the backend and frontend.



But there are differences...

Differences Between Client and Server


Model





Datastore classes, attributes, and methods—are only executed by the server.



Client code can access the model. But the code executes on the server.

Differences Between Client and Server


Data Provider (ds)





The data provider is a reference to the application’s set of data,

as well as the datastore class’s information and methods.


The ds object exists in both the server and client (WAF) code.


The ds object on the client is just a proxy of the datastore on the server.


Using ds from within the client-executed code will initiate AJAX calls to the server.

Differences Between Client and Server


Entities and Enity Collections



The concept of entity collections and entities are identical

whether your code is server-side or client-side.



There are however some differences in the syntax of commands client and server.

Differences Between Client and Server


Synchronous vs. Asynchronous



Server

var companyCollection = ds.company.query("name = :1", queryString);



Client

ds.company.query("name = :1", queryString, { 
   onSuccess: function(event) {
      ...   
   } ,
   onError : function(error) {
      ...
   }
});
